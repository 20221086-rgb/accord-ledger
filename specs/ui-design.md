# Accord Ledger - UI Design Guide

## 1. 디자인 컨셉

### 1.1 디자인 철학
**"따뜻한 신뢰감으로 갈등을 해결하다"**

Accord Ledger는 갈등 상황을 다루는 민감한 플랫폼입니다. 찬 느낌의 기술적 인터페이스보다는 **따뜻하고 부드러운 파스텔 톤**으로 사용자에게 안정감을 제공합니다.

### 1.2 컬러 팔레트

#### 기본 색상 (Primary Colors)
```
파스텔 핑크 (Primary)      #FFB6D9  (RGB: 255, 182, 217)
라이트 핑크 (Secondary)    #FFD1DC  (RGB: 255, 209, 220)
따뜻한 핑크 (Accent)      #FFC0D9  (RGB: 255, 192, 217)
```

#### 배경 색상 (Background)
```
오프화이트 (Main BG)       #FFF5F8  (RGB: 255, 245, 248)
라이트 그레이 (Sub BG)    #F5E6E8  (RGB: 245, 230, 232)
카드 배경 (Card BG)       #FFFBFC  (RGB: 255, 251, 252)
```

#### 텍스트 색상 (Text)
```
따뜻한 갈색 (Primary Text) #5A4A42  (RGB: 90, 74, 66)
중간 갈색 (Secondary Text) #8B7B73  (RGB: 139, 123, 115)
라이트 갈색 (Hint Text)   #B5A9A1  (RGB: 181, 169, 161)
```

#### 상태 색상 (Status)
```
성공 (Success)    #B5E7A0  (RGB: 181, 231, 160) - 파스텔 그린
경고 (Warning)    #FFE5B4  (RGB: 255, 229, 180) - 파스텔 오렌지
오류 (Error)      #FFB6C1  (RGB: 255, 182, 193) - 라이트 핑크
정보 (Info)       #B0E0E6  (RGB: 176, 224, 230) - 파스텔 블루
```

### 1.3 타이포그래피

#### 폰트 패밀리
- **헤딩**: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif (굵음)
- **본문**: 'Segoe UI', Tahoma, Geneva, Verdana, sans-serif (일반)
- **코드**: 'Courier New', Courier, monospace

#### 폰트 크기
```
H1 (페이지 제목)      32px  (font-weight: bold)
H2 (섹션 제목)        24px  (font-weight: bold)
H3 (소제목)           20px  (font-weight: 600)
본문 (Body)           16px  (font-weight: 400)
작은 본문 (Small)     14px  (font-weight: 400)
라벨 (Label)          12px  (font-weight: 500)
```

### 1.4 디자인 요소

#### 버튼 스타일
```
Primary Button (주요 액션)
- 배경: #FFB6D9 (파스텔 핑크)
- 텍스트: #FFF5F8 (오프화이트)
- 호버: #FFC0D9 (따뜻한 핑크)
- 그림자: rgba(255, 182, 217, 0.3)

Secondary Button (보조 액션)
- 배경: #F5E6E8 (라이트 그레이)
- 텍스트: #5A4A42 (따뜻한 갈색)
- 호버: #FFD1DC (라이트 핑크)
- 그림자: rgba(245, 230, 232, 0.3)

Disabled Button
- 배경: #E8D8D8 (회색)
- 텍스트: #B5A9A1 (라이트 갈색)
- 커서: not-allowed
```

#### 입력 필드 스타일
```
기본 상태
- 배경: #FFFBFC (카드 배경)
- 테두리: 1px solid #FFD1DC (라이트 핑크)
- 텍스트: #5A4A42 (따뜻한 갈색)

포커스 상태
- 배경: #FFFBFC
- 테두리: 2px solid #FFB6D9 (파스텔 핑크)
- 그림자: 0 0 8px rgba(255, 182, 217, 0.3)

에러 상태
- 배경: #FFFBFC
- 테두리: 2px solid #FFB6C1 (라이트 핑크/오류)
- 텍스트: #FFB6C1 (에러 메시지)
```

#### 카드 스타일
```
카드 배경: #FFFBFC (카드 배경)
테두리: 1px solid #FFD1DC (라이트 핑크)
그림자: 0 2px 8px rgba(255, 182, 217, 0.15)
보더 라디우스: 8px
패딩: 16px
```

#### 아이콘
```
크기: 24px (기본)
색상: #5A4A42 (따뜻한 갈색)
호버 색상: #FFB6D9 (파스텔 핑크)
사용: Google Material Icons (파스텔 톤으로 커스터마이징)
```

### 3.6 Agreement 기능
이 MVP에서는 Agreement 작성 및 서명 기능을 포함하지 않습니다. 
대신 Record와 Perspective 중심으로 감정/관계 기록을 안전하게 저장하고 검증하는 데 집중합니다.

### 3.7 기록 무결성 확인 패널

#### 목적
Record Detail 화면 내에서 Record 무결성 정보를 보조적으로 제공

#### 레이아웃
```
┌─────────────────────────────────────────────────────────┐
│  [◀ 돌아가기]                                           │
│                                                         │
│  ┌───────────────────────────────────────────────────┐ │
│  │ 기록 무결성 확인                                 │ │
│  │ ────────────────────────────────────────────── │ │
│  │ 이 기록은 SHA-256 해시 체인으로 연결되어 있어      │ │
│  │ 변조 여부를 확인할 수 있습니다.                  │ │
│  │                                                   │ │
│  │ [펼치기]                                        │ │
│  └───────────────────────────────────────────────────┘ │
│                                                         │
│  ┌───────────────────────────────────────────────────┐ │
│  │ Chain Valid                                       │ │
│  │ Record Hash                                       │ │
│  │ Latest Hash                                       │ │
│  │ Previous Hash                                     │ │
│  │ Tamper Detected                                   │ │
│  │ Last Verified At                                  │ │
│  └───────────────────────────────────────────────────┘ │
```

#### 주요 요소
- [ ] 뒤로 가기 버튼
- [ ] 전체 검증 상태 (✓ PASS 또는 ✗ FAIL)
- [ ] 상태에 따른 메시지
  - 통과: "모든 기록이 안전합니다"
  - 실패: "데이터 변조가 감지되었습니다"
- [ ] 검증 요약 (총 항목, 완료, 실패, 성공률)
- [ ] 상세 검증 결과
  - 각 항목의 타입, 작성자 (해당시)
  - 해시 값 (축약)
  - 체인 상태
  - 개별 검증 상태
- [ ] 변조 시 경고 및 오류 표시
